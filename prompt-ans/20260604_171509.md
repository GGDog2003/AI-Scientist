这次根因已经非常明确：你把 `OPENAI_API_KEY` 设置成了中文占位文本 `你的新 key`，不是真实 API Key。HTTP 请求头只能接受 ASCII，底层 `httpx` 在构造 `Authorization` 头时就直接炸成了 `UnicodeEncodeError`，所以这不是模型本身的问题，也不是工作流逻辑的问题。

我已经把 [app/config.py](/D:/python/AI-Scientist/app/config.py) 和 [app/main.py](/D:/python/AI-Scientist/app/main.py) 补强了。现在在真正发请求前，程序会先检查 API Key 是否包含非 ASCII 字符、是否像占位文本；如果你再填 `你的新 key` 这种值，它会在启动前直接报可读错误，而不是再抛底层长堆栈。

你现在应该把这行：

```powershell
$env:OPENAI_API_KEY="你的新 key"
```

换成真实平台签发的 key，例如：

```powershell
$env:OPENAI_API_KEY="sk-xxxxxx"
```

然后再启动：

```powershell
python main.py start --topic "MRI图像超分辨率" --domain "人工智能" --paper-dir "C:\Users\GGDog\Desktop\testagent"
```

改了什么：增加了启动前的 API Key 合法性校验，并把 ASCII 编码失败错误翻译成直接可执行的提示。
