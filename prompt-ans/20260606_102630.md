# 本次修改说明

已把真实 LLM 主链路改成你要的方向：

1. 去掉了 `stub` 模式。
   `app/config.py`、`src/ai_scientist/config.py`、`app/agents/base_agent.py`、`src/ai_scientist/agents.py` 都不再保留 `use_stub_agent`、`AI_SCIENTIST_USE_STUB` 和 `_stub_response` 分支，只允许真实大模型运行。

2. 把方法级提示词做成统一管理。
   `app/prompts/__init__.py` 和 `src/ai_scientist/prompts.py` 新增统一的角色系统提示词与 `build_task_prompt()`，每次 agent 方法调用前都会按 `task_name` 注入对应提示词。

3. 把实验结果分流硬规则接入真实 LLM 链路。
   `app/workflows/state_machine.py` 和 `src/ai_scientist/workflow.py` 里新增了基于 `PSNR/SSIM` 与总结关键词的强制分流：
   - `PSNR >= 40` 或 `SSIM >= 0.995`：`direct_to_writing`
   - `PSNR >= 34` 或 `SSIM >= 0.97`：`minor_revision`
   - 否则：`major_revision`
   导师 agent 在这个节点只能补评论，不能改写结论。

4. 把实验结果决策改成强枚举。
   `app/schemas/experiment_result.py` 和 `src/ai_scientist/schemas.py` 里新增 `ExperimentGateDecision`，避免真实 LLM 再返回乱值把流程打偏。

5. 保留并固化流程上限。
   创新点导师最多 3 轮，之后强制进入实验；论文导师最多 5 轮，之后强制进盲审。
