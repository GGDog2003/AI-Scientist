**改了什么**
- 保留了现有 CUI，不替换原来的 [app/main.py](/D:/python/AI-Scientist/app/main.py) 链路，只额外增加了 GUI 专用桥接层：[app/gui_api.py](/D:/python/AI-Scientist/app/gui_api.py) 和 [app/gui_cli.py](/D:/python/AI-Scientist/app/gui_cli.py)。GUI 通过这层调用现有工作流的 `start/resume/state` 能力，返回纯 JSON，避免解析终端文本。

  - 初始只有“导入论文目录”“开始撰写”两个主按钮。
  - 未导入论文目录前，“开始撰写”不可点击。
  - 启动后按进度列表展示当前链路状态。
  - 列表项若对应产出的 `md/json` 文件，可点击打开。
  - 中断等待实验结果时显示“导入实验结果”按钮，并在确认后触发 resume。
  - 全部完成后显示“阅读最终论文成稿”按钮并打开最终论文。
- 新增了 Tauri Rust 后台桥接：[ui/src-tauri/src/lib.rs](/D:/python/AI-Scientist/ui/src-tauri/src/lib.rs)。它负责：
  - 启动 Python GUI CLI。
  - 轮询线程状态。
  - 恢复挂起流程。
  - 打开工作区里的产物文件。

**还剩什么风险**
- 当前机器缺少 Rust toolchain，所以 [ui/src-tauri](/D:/python/AI-Scientist/ui/src-tauri) 还没做真机构建验证；安装 `rustup` / `cargo` 后才能跑 `npm install`、`npm run tauri dev` 做完整联调。

- 我没有清理仓库里已有的 `__pycache__` 未跟踪文件，因为那不是这次 GUI 功能本身的一部分，也避免误删你的现有状态。
