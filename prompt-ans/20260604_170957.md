这次问题不是论文目录，也不是工作流本身先挂了，根因有两个，而且是叠加的。

第一，你在 PowerShell 里用了 `set`。在 PowerShell 里它不是给子进程设置环境变量的正确方式，应该用 `$env:...`。所以你虽然输入了 `set OPENAI_MODEL_NAME=deepseek-v4-flash`，但 Python 进程实际上没拿到这个值，程序才会打印 `[AI-Scientist] 运行模式: llm(openai:gpt-4.1-mini)`，说明它仍然在用默认 OpenAI 模型，而不是你想要的 DeepSeek。

第二，项目原来只识别 `AI_SCIENTIST_MODEL`，不兼容你这种 `OPENAI_MODEL_NAME` 写法。我已经把 [app/config.py](/D:/python/AI-Scientist/app/config.py) 改了，现在会兼容 `OPENAI_MODEL_NAME`，并且在启动时打印“模型配置来源”；同时在 [app/main.py](/D:/python/AI-Scientist/app/main.py) 里补了更明确的异常提示，如果再次命中 `unsupported_country_region_territory`，会直接提醒你检查 PowerShell 变量写法和模型来源。依赖里我也补上了 `langchain-openai`，分别更新了 [requirements.txt](/D:/python/AI-Scientist/requirements.txt) 和 [pyproject.toml](/D:/python/AI-Scientist/pyproject.toml)。

你现在应该这样启动：

```powershell
$env:OPENAI_BASE_URL="https://api.deepseek.com"
$env:OPENAI_API_KEY="你的key"
$env:OPENAI_MODEL_NAME="deepseek-v4-flash"
python main.py start --topic "MRI图像超分辨率" --domain "人工智能" --paper-dir "C:\Users\GGDog\Desktop\testagent"
```

启动后先看两行：

```powershell
[AI-Scientist] 运行模式: llm(openai:deepseek-v4-flash)
[AI-Scientist] 模型配置来源: OPENAI_MODEL_NAME
```

只有出现这两行，才说明这次真的读到了你的 DeepSeek 模型配置；如果还是 `default` 或 `gpt-4.1-mini`，那就说明环境变量仍然没传进去。

我做了两类验证。第一，`python -m compileall main.py app tests` 已通过。第二，我尝试直接导入配置验证运行逻辑，但当前这个 Codex 沙箱环境本身没装 `langgraph`，所以没法在这里做完整运行级验证。

剩余风险有两个。第一，你贴出来的 `deepseek-v4-flash` 是否是你账号下当前可调用的真实模型名，我这里没有替你在线核验；如果模型名本身不对，后面还会报模型不存在或接口参数错误。第二，你贴图里的 key 已经泄露，应该立刻去对应平台作废并换新 key。
