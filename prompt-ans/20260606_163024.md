已修复这类“点击导入论文目录没有任何反应”的前端根因，核心改动在 [App.vue](/D:/python/AI-Scientist/ui/src/App.vue)、[tauri.conf.json](/D:/python/AI-Scientist/ui/src-tauri/tauri.conf.json) 和 [default.json](/D:/python/AI-Scientist/ui/src-tauri/capabilities/default.json)。

改了什么

- 给 Tauri 主窗口显式补了 `label: "main"`，避免 capability 绑定不到窗口。
- 新增了 `ui/src-tauri/capabilities/default.json`，授予主窗口 `core:default` 和 `dialog:default` 权限，允许前端调用 `@tauri-apps/plugin-dialog` 的 `open()`。
- 重写了 [App.vue](/D:/python/AI-Scientist/ui/src/App.vue) 为正常 UTF-8 版本，之前这个文件已经有明显乱码；同时给“导入论文目录”和“导入实验结果”都加了 `try/catch`，如果对话框权限或调用失败，会直接在界面底部显示错误，而不是看起来像没反应。

有没有验证

- 已执行 `npm run build`，前端构建通过。
- 尝试执行 `cargo check` 做 Tauri 侧校验，但当前终端环境里 `cargo` 不在 PATH，所以这一步没法完成。

还剩什么风险

- `capability` 配置只有在重新启动 Tauri 进程后才会生效。如果你之前已经开着 `npm run tauri dev`，需要先停掉再重新启动。
- 如果重启后仍然没弹窗，就不是前端按钮问题了，而要继续查本机 Tauri/Rust 环境或插件版本联动。
