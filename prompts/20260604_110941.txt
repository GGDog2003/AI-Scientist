[datawhalechina/hello-agents](https://github.com/datawhalechina/hello-agents) 请根据hello-agents的规范，给我罗列下面需求的实现细节，放在/实现方式目录中以markdown存放

我要实现一个科研agent，实现以下功能：
本agent的目标是一个自主读文献，写代码，写论文的多agent共同协作的agent；本agent分为以下几个角色：研究生、导师、审稿人。每个角色都是一个子agent。
一、研究生
自行阅读论文
归纳每篇论文的创新点，不足点
每篇论文如果是基于别人论文改良的论文，自行阅读基座论文，并说明提升点
读完10篇论文之后，自行汇总创新点成md文档
根据汇总的md文档，筛选最有可能提升的创新点，并在“开组会”的时候告诉agent导师
如果导师有修改建议，就继续修改，直到导师方面通过此创新点方可开始实验
根据创新点设计实验
如果是人工智能相关领域，设计python代码实验
等待人类手动执行实验，并告诉agent实验结果。根据实验结果初步判断怎么去构思论文话术，汇总成md文档
根据初步构思的论文话术，撰写学术论文。论文方法、数据不能造假，要严格按照人类执行的实验结果撰写
每一版本的论文撰写结束后，请研究生自行进行一轮审核，审核无误交给agent导师二轮审核

二、agent导师
导师复制的就是创新点的把关，实验结果的评判，以及论文的二轮审查和修改。具体有以下要求：
对于创新点：
如果是基于基座论文的修改，添加进去的模块、方法是否足够新颖，是否足够能做出好实验效果？
如果是自行创新的论文，那创新点是否足够新颖，是否已经有他人做过此类方法？
基于自己的判断，给出学生创新点修改建议。如果没有，就通过该创新点，让研究生去做实验
对于论文：
审核论文整体结构是否符合学术规范，摘要、引言、相关工作、方法、实验、结论、参考文献等章节是否完整且逻辑连贯
重点审查方法部分的技术描述是否准确、清晰，公式推导是否正确，实验设计是否科学合理
评估实验结果的分析是否深入，是否能准确解释实验现象，是否与创新点形成有效呼应
检查论文是否存在学术不规范问题，包括但不限于引用不当、数据表述模糊、结论夸大等
针对论文的语言表达、逻辑漏洞、内容缺失等问题给出具体修改意见，要求研究生逐条修改
二轮审核通过后，将论文提交给 agent 审稿人进行三轮盲审
对于实验结果：
评估实验结果是否验证了提出的创新点，实验数据是否具有统计学意义
判断实验是否存在对照组缺失、参数设置不合理、评估指标不全面等问题
若实验结果未达预期，与研究生共同分析原因，提出实验改进方案或创新点调整建议
确认实验结果的真实性和可复现性，严禁任何形式的数据造假或结果美化
三、agent 审稿人
审稿人以第三方客观中立的视角，模拟顶级学术会议 / 期刊的审稿标准，对论文进行严格的盲审，提出专业、尖锐且具有建设性的评审意见。具体要求如下：
论文创新性评审
评估论文的核心创新点在该研究领域的贡献程度，区分增量式改进和突破性创新
对比最新相关研究成果，指出论文中哪些内容是已有工作的重复，哪些是真正的原创性贡献
判断创新点的学术价值和应用前景，是否能推动该领域的发展
论文科学性评审
审查研究方法的科学性和严谨性，实验设计是否能够有效验证研究假设
检查实验数据集的来源、规模、划分方式是否合理，是否存在数据泄露问题
评估实验结果的分析是否全面，是否讨论了方法的局限性和未来改进方向
验证论文中所有引用的文献是否准确，是否存在过度自引或不相关引用
论文写作质量评审
评价论文的整体写作水平，语言是否流畅、专业，术语使用是否规范
检查图表是否清晰、规范，标注是否完整，是否能独立说明问题
指出论文中存在的逻辑混乱、表述不清、前后矛盾等问题
评估论文的可读性和易懂性，是否适合目标学术期刊 / 会议的读者群体
评审结果输出
给出明确的评审结论：接受、小修后接受、大修后接受、拒稿
针对上述评审维度，逐条列出详细的评审意见和修改建议
若为大修或拒稿，明确说明主要原因，并给出具体的改进方向
若为小修或接受，指出需要完善的细节问题
四、多 Agent 协作流程规范
文献调研阶段：研究生 Agent 独立完成 10 篇相关领域顶会 / 顶刊论文的阅读与分析，生成《文献调研汇总报告.md》，包含每篇论文的创新点、不足点、基座论文对比及整体研究趋势分析
创新点讨论阶段：研究生 Agent 基于文献调研提出 3-5 个潜在创新点，在 "组会" 中向导师 Agent 汇报；导师 Agent 逐一评估并给出修改意见，双方迭代直至确定最终创新点
实验设计与执行阶段：研究生 Agent 根据确定的创新点设计详细的实验方案，生成《实验设计方案.md》，包含实验环境、数据集、评估指标、代码框架等内容；人类用户手动执行实验后，将实验结果反馈给研究生 Agent
论文撰写阶段：研究生 Agent 根据实验结果撰写论文初稿，完成自审后提交给导师 Agent；导师 Agent 进行二轮审核并给出修改意见，研究生 Agent 修改后提交给审稿人 Agent
论文评审阶段：审稿人 Agent 进行三轮盲审并输出评审意见；若为大修 / 拒稿，返回研究生 Agent 修改后重新进入导师审核环节；若为小修 / 接受，研究生 Agent 完成最终修改后生成终稿
版本管理规范：所有文档（文献报告、实验方案、论文各版本、评审意见）均按 "日期_版本号_文档名称.md" 格式命名，保存在对应目录下，保留完整修改历史
五、技术实现要求
所有 Agent 均基于大语言模型实现，支持上下文记忆和多轮对话
文献阅读功能需集成 PDF 解析工具，能够提取论文文本、图表、公式等内容
代码生成功能需支持 Python 语言，能够生成可运行的实验代码框架，并包含详细的注释
系统需提供文件读写接口，能够自动生成和保存 markdown 格式的文档
实现 Agent 间的消息传递机制，确保协作流程的自动化和连贯性
保留人类干预接口，允许用户在任何阶段介入并修改 Agent 的输出结果
