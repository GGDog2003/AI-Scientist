# 实验设计方案

## 实验目标

验证在MRI图像超分辨率任务中，所提出方法是否能在保证解剖结构一致性与临床可用性的前提下，优于经典插值方法、卷积网络方法与近期生成式/Transformer类方法；同时分析不同退化建模、损失设计与2D/2.5D/3D建模策略对性能的影响。说明：当前工作区未发现可读取的本地论文/历史文档，因此以下方案为通用且可执行的实验设计框架，不包含任何虚构实验结果。

## 数据集

- fastMRI（膝关节/脑MRI，如本地可得）
- IXI（T1/T2/PD，多模态脑MRI）
- BraTS（若研究脑部结构保持，可作为补充）
- Clinical/Institutional MRI dataset（若课题组自有数据可用，需补充伦理与划分说明）
- 可选：NAMIC、Human Connectome Project等公开脑MRI数据集（按可获得性决定）

## Baseline

- Bicubic / B-spline interpolation
- SRCNN / VDSR
- EDSR / RDN / RCAN
- SwinIR 或其他Transformer-based SR方法
- ESRGAN / Real-ESRGAN风格感知型方法（若关注视觉质量）
- 针对医学影像的MRI SR方法：如基于3D CNN、残差密集网络、注意力网络的方法（以复现到的公开实现为准）
- 若方法含物理先验：与仅图像域SR、仅k-space约束方法分别对比

## 评价指标

- PSNR
- SSIM
- NRMSE/RMSE
- HFEN（高频误差范数）
- LPIPS（若评估感知质量，可选）
- Edge preservation / Gradient correlation（边缘保持，可选）
- 推理时间、参数量、FLOPs
- ROI级指标：白质/灰质、病灶区域或结构边界区域上的PSNR/SSIM
- 下游任务指标：如分割Dice、分类AUC的变化（若要证明临床/任务价值）
- 统计显著性检验：配对t检验或Wilcoxon signed-rank test

## 消融实验

- 退化模型消融：固定双三次下采样 vs 更真实的MRI退化（模糊+噪声+降采样）
- 输入维度消融：2D vs 2.5D（相邻切片堆叠）vs 3D
- 网络模块消融：去掉注意力模块、频域模块、跨尺度融合模块、数据一致性模块（按你的模型实际组件替换）
- 损失函数消融：L1/L2 vs SSIM loss vs perceptual loss vs edge/gradient loss vs frequency loss
- 训练策略消融：是否使用自监督/预训练/迁移学习
- 上采样倍率消融：×2、×4（必要时×3）
- 噪声鲁棒性消融：不同SNR条件下测试
- 跨域泛化消融：训练于一个数据集/模态，测试于另一数据集/模态
- patch大小与batch size消融
- 若有生成式模块：是否使用对抗损失/扩散先验的消融

## 代码模块

- datasets/mri_sr_dataset.py：数据读取、病人级划分、2D/2.5D/3D采样
- preprocessing/mri_preprocess.py：归一化、重采样、裁剪、配准接口
- degradation/mri_degradation.py：bicubic与MRI真实退化模拟
- models/baselines_wrapper.py：统一封装SRCNN/VDSR/EDSR/RCAN/SwinIR等baseline
- models/your_model.py：主模型实现
- losses/sr_losses.py：L1、SSIM、perceptual、edge、frequency等损失
- trainers/sr_trainer.py：训练、验证、早停、日志记录
- evaluation/metrics.py：PSNR、SSIM、NRMSE、HFEN、LPIPS等指标
- evaluation/statistics.py：显著性检验与置信区间
- evaluation/downstream_eval.py：分割/分类下游任务评估接口
- scripts/run_train.py：训练入口
- scripts/run_test.py：测试入口
- scripts/run_ablation.py：批量消融实验脚本
- scripts/prepare_splits.py：病人级数据划分
- utils/visualization.py：误差图、ROI放大图、频谱可视化
- utils/reproducibility.py：随机种子、环境记录、配置保存

## 人工执行步骤

1. 明确任务设定：单模态MRI超分辨率，指定倍率（建议×2、×4）以及输入输出空间分辨率。
2. 确定数据来源与协议：选择公开数据集，完成训练/验证/测试划分，避免病人级泄漏。
3. 定义低分辨率生成流程：优先建立两套协议——理想退化（bicubic）与更真实MRI退化（模糊、噪声、降采样，必要时模拟k-space欠采样后重建）。
4. 统一预处理：N4 bias correction（如需要）、配准（如需要）、裁剪、强度归一化、去除空白背景、重采样到统一spacing。
5. 实现或复现所有baseline，统一训练轮数、优化器、学习率计划、数据增强和early stopping策略。
6. 训练主方法：至少运行3个随机种子，记录均值与标准差。
7. 在验证集上调参与模型选择，不得使用测试集调参。
8. 在测试集上评估所有方法，分别报告整体指标、不同倍率、不同模态、不同器官/区域结果。
9. 完成消融实验：逐一移除关键模块或替换关键设计，保持其余设置不变。
10. 完成泛化实验：跨数据集、跨扫描参数、跨噪声水平测试。
11. 完成稳健性分析：不同退化强度、不同切片厚度、不同样本量下的性能变化。
12. 完成效率分析：训练/推理耗时、显存占用、参数量、FLOPs。
13. 如条件允许，增加下游任务验证：将SR结果输入分割/检测模型，衡量是否提升下游性能。
14. 进行可视化：展示代表性切片、误差图、边缘区域放大图、频谱对比。
15. 进行统计检验并撰写误差分析：分析失败案例，如纹理幻觉、边界过锐化、病灶失真。
16. 整理实验日志、配置文件、随机种子、环境依赖，保证可复现。
